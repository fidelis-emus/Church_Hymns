# Worship Portal & Hymnal Display PHP REST API (`api.php`)

Welcome to your lightweight, shared-hosting optimized, full-featured **PHP REST API**. This entire backend is encapsulated within a single file (`api.php`), leveraging a local, serverless **SQLite** database (`/storage/database.sqlite`). It replaces the previous multi-dependency Node.js/PostgreSQL container architecture with an elegant, ultra-performant, single-file PHP solution.

---

## 🚀 Architectural Design & Key Features

1. **Pure PHP 8+ Implementation**: Built completely in vanilla PHP with no heavy frameworks, making it extremely lightweight and compatible with standard shared hosting environments.
2. **Auto-Initializing SQLite Storage**: The API automatically creates the `/storage/` directory and `/storage/database.sqlite` file, runs table schema migrations, and seeds default administrators, templates, and configurations on the very first request.
3. **Prepared Statements**: Uses PHP Data Objects (PDO) prepared statements exclusively to prevent SQL injection vulnerabilities.
4. **Stateless HMAC-SHA256 Token Auth**: Employs a lightweight, standard-compliant, self-signed stateless JWT implementation (HMAC-SHA256 signature verification) for secure admin sessions without database lookups.
5. **CORS & Options Preflight Support**: Complete Cross-Origin Resource Sharing (CORS) header coverage to enable seamless consumption by decoupled frontends (such as React, Vue, Angular, or Mobile Apps).
6. **Robust Input Validation & Error Handling**: Direct JSON-format error responses with standard-compliant HTTP Status Codes (`200`, `201`, `400`, `401`, `403`, `404`, `500`).
7. **Integrated Logging**: Automatically logs all incoming requests, status codes, source IPs, and executing users to a local flat file (`/storage/api_requests.log`).

## 📁 File Structure & Requirements

Ensure your hosting directory looks as follows:
```text
your-root-folder/
├── api.php             # Core API Engine (All Endpoints & Logic)
├── .htaccess           # Clean REST URL Router configuration (Apache)
└── storage/            # Created automatically by api.php
    ├── database.sqlite # Autogenerated SQLite database
    └── api_requests.log# Autogenerated request audit log
```

### Server Requirements
- **PHP Version**: `8.0` or higher
- **PDO Extensions**: `pdo_sqlite` enabled
- **File Permissions**: Write access allowed for the directory containing `api.php` so the `storage` folder can be created.

## 🔑 Default Credentials & Authentication
The database is auto-seeded with a default Super Administrator:
- **Username**: `admin`
- **Password**: `admin123`
- **Role**: `SUPER_ADMIN`

To perform operations on protected routes, first authenticate via `POST /auth/login` to obtain your stateless Bearer Token, and include it in your HTTP Headers:
```http
Authorization: Bearer <your_token_here>
```

## 📡 REST API Reference

All requests must set `Content-Type: application/json` and submit payloads as JSON objects. All responses are returned as formatted JSON.

#### `POST /auth/login`
Authenticates user credentials and issues a stateless token.
- **Payload**:
  ```json

    "username": "admin",
    "password": "admin123"
  }
  ```
- **Response (200 OK)**:
  ```json

    "user": {
      "id": 1,
      "username": "admin",
      "email": "admin@church.org",
      "role": "SUPER_ADMIN"
  ```

Fetches profile information of the currently authenticated token owner.


### 2. Users Management (Super Admin Only)

#### `GET /auth/users` *(Protected)*
- **Response (200 OK)**:
  ```json
  [
    {
      "id": 1,
      "username": "admin",
      "email": "admin@church.org",
      "role": "SUPER_ADMIN",
      "created_at": "2026-07-11 12:00:00"
    }
  ]
  ```

#### `POST /auth/users` *(Protected)*
Creates a new administrator account.
- **Payload**:
  ```json
  {
    "username": "pastor_john",
    "email": "john@church.org",
    "password": "securepassword",
    "role": "ADMIN"
  }
  ```

#### `PUT /auth/users/{id}` *(Protected)*
Updates an existing user's information or resets their password.

#### `DELETE /auth/users/{id}` *(Protected)*
Removes an administrator account. Self-deletion is protected.

### 3. Hymnal Display Endpoints

#### `GET /hymns`
Lists and searches hymns with pagination, category filtering, and sorting.
- **Query Parameters**:
  - `page`: Page index (default: `1`)
  - `limit`: Records per page (default: `10`)
  - `category`: Filter by category (e.g. `Faith`, `Grace`, `Praise`)
  - `search`: Keywords to search across hymn numbers, titles, lyrics, and choruses.
  - `sortBy`: Sort field (`hymn_number`, `title`, `category`)
  - `sortOrder`: Sort direction (`ASC` or `DESC`)
- **Response (200 OK)**:
  ```json
  {
    "total": 12,
    "page": 1,
    "limit": 10,
    "pages": 2,
    "hymns": [
      {
        "id": 1,
        "hymnNumber": 101,
        "title": "Amazing Grace",
        "lyrics": "Amazing grace how sweet the sound...",
        "chorus": "Was lost but now am found...",
        "category": "Grace",
        "language": "English"
      }
    ]
  }
  ```

#### `POST /hymns` *(Protected)*
Adds a new hymn to the registry.

#### `PUT /hymns/{id}` *(Protected)*
Updates lyrics, title, or details of an existing hymn.

#### `DELETE /hymns/{id}` *(Protected)*
Deletes a hymn from the registry.

#### `POST /hymns/{id}/duplicate` *(Protected)*
Duplicates an existing hymn, auto-assigning the next available highest hymn number.

### 4. Interactive Live Projection Display

The application supports pushing active items to a live projection screen or mobile companion web application in real-time.

#### `GET /display/current`
Retrieves the active slides, hymns, scripture readings, or announcements currently selected to be shown on church displays or members' screens.
- **Response (200 OK)**:
  ```json
  {
    "id": 1,
    "displayType": "HYMN",
    "recordId": 12,
    "lastUpdated": "2026-07-11 15:30:22",
    "status": "PUBLISHED",
    "data": {
      "id": 12,
      "hymnNumber": 101,
      "title": "Amazing Grace",
      "lyrics": "Amazing grace how sweet the sound...",
      "chorus": "Was lost but now am found...",
      "category": "Grace"
    }
  }
  ```

#### `POST /display/hymn/{id}` *(Protected)*
Publishes a selected hymn to the live screen.

#### `POST /display/announcement/{id}` *(Protected)*
Publishes an announcement slide to the live screen.

#### `POST /display/citation/{id}` *(Protected)*
Publishes a Scripture Reading to the live screen.

#### `POST /display/message/{id}` *(Protected)*
Publishes custom message slides to the live screen.

#### `POST /display/welcome` *(Protected)*
Resets the projection screen to the default "Welcome Slide".

#### `GET /display/history` *(Protected)*
Fetches a list of the 20 most recently displayed slides for operator audit.

### 6. System Settings & Utilities

#### `GET /settings`
Fetches active branding colors, titles, and display styles.

#### `POST /settings` *(Protected)*
Saves branding parameters.

#### `GET /logs` *(Protected)*
Fetches the 100 most recent system operator actions from the database.

#### `GET /stats` *(Protected)*
Fetches high-level metrics of the system, recent histories, and active logs.

#### `GET /backup` *(Protected)*
Generates and downloads a complete, structured JSON database dump of all tables, configurations, and users for effortless offline backup.

#### `POST /restore` *(Protected)*
Accepts a structured JSON database backup file and completely restores database tables, indices, credentials, and settings.

#### `GET /health`
Returns system status. Great for uptime trackers and setup verifications.

## 🌐 Deploying to Shared Hosting (cPanel / Apache)

1. Upload `api.php` and `.htaccess` to your server's web root directory (e.g., `public_html/` or a subdirectory `api/`).
2. Verify that your server allows `.htaccess` rewrites (`AllowOverride All` configured in Apache).
3. If using Nginx instead of Apache, add the following directive to your site configuration block:
   ```nginx
   location / {
       try_files $uri $uri/ /api.php?$query_string;
   }
   ```
4. Access `http://yourdomain.com/health` to confirm successful setup!
